ncdu

1. ncdu (NCurses Disk Usage)** это консольный анализатор использования диска с интерактивным интерфейсом.​

Если на сервере внезапно закончилось место, ncdu просканирует файловую систему и покажет вам красивый список папок, отсортированный по размеру. Вы сможете ходить по ним стрелочками (как в файловом менеджере), проваливаться внутрь и удалять гигантские логи или старые бэкапы нажатием одной кнопки.

2. Установка и первый запуск​

Утилита не идет в Linux по умолчанию, но весит копейки и ставится за пару секунд:

  • Ubuntu / Debian: sudo apt install ncdu
  • CentOS / RHEL: sudo yum install epel-release && sudo yum install ncdu
  • Alpine (Docker): apk add ncdu

Как запустить?​

Есть два основных сценария запуска:

Вариант А: Анализ конкретной папки Перейдите в подозрительную папку (например, с логами) и просто введите:

ncdu

Утилита просканирует только текущую папку и всё, что внутри неё.

Вариант Б: Глобальный поиск по всему серверу Если вы не знаете, где спрятался огромный файл, сканируйте корень диска (/). Важно: для сканирования системных папок нужны права администратора!

sudo ncdu /

Сначала появится окно сканирования (подсчет файлов), на больших дисках это может занять минуту. Как только сканирование завершится, откроется интерактивный интерфейс.

3. Как пользоваться (Горячие клавиши)​

Интерфейс интуитивно понятен. Слева указан размер, затем графическая шкала (решетки #), а справа — название папки или файла.

Управление происходит с клавиатуры:

  • ⬆️ Стрелка Вверх / ⬇️ Вниз — Перемещение по списку.
  • ➡️ Стрелка Вправо (или Enter) — Провалиться внутрь выбранной папки.
  • ⬅️ Стрелка Влево — Выйти на уровень назад (в родительскую папку).
  • 🗑️ d (Delete) — УДАЛИТЬ выбранный файл или папку. Система попросит подтверждения (кнопки Yes / No / Don't ask me again).
  • Сортировка:
    • s — Отсортировать по размеру (Size).
    • n — Отсортировать по имени (Name).
  • q (Quit) — Выйти из программы обратно в обычный терминал.

4. Особенности​

Если вы работаете со сложными серверами, вам пригодятся эти ключи запуска:

1. Флаг -x (Только текущий диск)​

Бывает, что к вашему Linux-серверу примонтирован внешний сетевой диск на 10 Терабайт (например, в папке /mnt/storage). Если вы запустите sudo ncdu /, он начнет сканировать и этот сетевой диск, что займет часы. Флаг -x заставляет программу сканировать только ту физическую файловую систему, на которой она была запущена, игнорируя примонтированные диски.

sudo ncdu -x /

2. Режим "Только для чтения" (-r)​

Если вы сканируете критически важный Production-сервер, есть страх случайно нажать клавишу d и удалить важную базу данных. Запустите ncdu в режиме Read-Only. В этом режиме функция удаления будет полностью отключена.

sudo ncdu -r /

3. Экспорт отчета в файл (-o)​

Если сервер еле дышит из-за нехватки места и вы не хотите держать окно терминала открытым, можно попросить ncdu собрать данные и выгрузить их в файл.

# Сохраняем результат в файл
sudo ncdu -1xo- / > /tmp/ncdu_report.txt

# Позже открываем этот файл для просмотра
ncdu -f /tmp/ncdu_report.txt

Резюме​

  • ncdu — лучший инструмент для ответа на вопрос "Куда делось место на диске?".
  • Запуск от корня: sudo ncdu / (или sudo ncdu -x /, чтобы игнорировать сетевые диски).
  • Управление стрелочками. Enter — войти, Стрелка влево — выйти.
  • Удаление файлов прямо из интерфейса кнопкой d.
  • Безопасный режим (без права на удаление) включается флагом -r. `
