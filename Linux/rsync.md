# Rsync: Синхронизация и копирование


**`rsync` (Remote Sync)** — стандартный инструмент в Linux для копирования и синхронизации файлов и директорий, он может работать как локально (на одном диске), так и по сети (через SSH).

Главная фишка `rsync` — **дельта-алгоритм (Delta Transfer)**. 
При повторном копировании он не передает файлы целиком. Он вычисляет разницу и передает по сети только измененные куски файлов (или докачивает то, что оборвалось при прошлой передаче). Это колоссально экономит время и трафик.

## 2. Разбор рабочего примера

Задача: стянуть папку с удаленного сервера на локальный компьютер, игнорируя тяжелые директории с кэшем и бэкапами.

```bash
rsync -avzP \
--exclude='backup' \
--exclude='cache' \
--exclude='managed_cache' \
--exclude='stack_cache' \
--exclude='tmp' \
example.ru@example.ru:/srv/www/example.ru/htdocs/main/ ./main/
```

### Расшифровка флагов (`-avzP`)
Эта комбинация используется в 99% случаев:
*   **`-a` (archive):** Архивирует. Рекурсивно копирует все вложенные папки, сохраняя оригинальные права доступа, владельцев и время изменения файлов.
*   **`-v` (verbose):** Выводит в терминал список того, что сейчас копируется.
*   **`-z` (compress):** Сжимает файлы на лету перед отправкой по сети и распаковывает на принимающей стороне.
*   **`-P` (progress + partial):** Показывает прогресс-бар для каждого файла и сохраняет частично скачанные файлы при обрыве соединения (для докачки).

### Исключения (`--exclude`)
Флаг `--exclude='имя_папки'` позволяет пропустить ненужные директории или файлы. Можно использовать маски, например: `--exclude='*.log'`.

## 3. Главная ловушка: Слеш на конце пути

Самая частая ошибка — неправильное использование слеша `/` в пути источника.

**Вариант А: БЕЗ слеша на конце**
```bash
rsync -a /var/www/src /backup/dest
```
👉 *Результат:* Скопирует **саму папку** `src`. Внутри `dest` появится папка `src` (путь будет `/backup/dest/src/`).

**Вариант Б: СО слешем на конце**
```bash
rsync -a /var/www/src/ /backup/dest
```
👉 *Результат:* Скопирует **содержимое** папки `src`. Все файлы высыпятся прямо в корень `/backup/dest/`.

> **Правило:** Слеш на конце означает слово *"содержимое"*.

## 4. Частые команды

### 1. Загрузка НА сервер (Upload)
Меняем местами источник и назначение:
```bash
rsync -avzP ./my_website/ root@192.168.1.10:/var/www/site/
```

### 2. Идеальное зеркалирование (`--delete`)
По умолчанию `rsync` только *добавляет* новые файлы. Если вы удалили файл локально, на сервере он останется, флаг `--delete` заставит `rsync` удалить на сервере всё, чего нет в источнике.
```bash
rsync -avzP --delete ./my_website/ root@192.168.1.10:/var/www/site/
```

### 3. Нестандартный порт SSH (`-e`)
Если на сервере SSH висит не на 22 порту (а, например, на 2222):
```bash
rsync -avzP -e 'ssh -p 2222' ./data/ admin@server.com:/backup/
```

### Резюме
*   `rsync` — идеален для бэкапов и деплоя статики. Возобновляет прерванные загрузки.
*   Стандартный набор флагов: `-avzP`.
*   Будьте предельно внимательны со слешем `/` на конце пути источника.
*   Используйте флаг `--dry-run` (или `-n`), чтобы сделать "сухой прогон" и посмотреть, что сделает команда, не изменяя файлы физически.
