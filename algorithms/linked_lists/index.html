Связанные списки (Linked Lists)

Связанный список — это структура данных, состоящая из узлов (nodes), где каждый узел хранит значение и указатель на следующий узел (в односвязном списке) или на следующий и предыдущий (в двусвязном).

В отличие от массивов (слайсов), элементы связанного списка не лежат в памяти последовательно. Это означает, что:

  • Вставка/удаление в начало или середину списка: O(1) (если мы уже находимся на нужном элементе).
  • Доступ к элементу по индексу: O(N) (приходится идти от головы по цепочке указателей).

В Go есть стандартный пакет container/list (двусвязный список), но на собеседованиях всегда требуют реализовать список и операции над ним вручную.

1. Базовая структура​

В Go односвязный список обычно описывается так:

type ListNode struct {
    Val  int
    Next *ListNode // Указатель на следующий узел
}

2. Разворот связанного списка (Reverse Linked List)​

Это, пожалуй, самая частая задача на собеседованиях (LeetCode #206). Нужно развернуть указатели так, чтобы первый элемент стал последним, а последний — первым. Сложность должна быть $O(N)$ по времени и $O(1)$ по памяти (in-place).

Итеративный подход (стандарт)​

Используем три указателя: prev (предыдущий), curr (текущий) и next (следующий, чтобы не потерять хвост).

func reverseList(head *ListNode) *ListNode {
    var prev *ListNode // Изначально предыдущего нет (nil)
    curr := head
    
    for curr != nil {
        // 1. Сохраняем ссылку на остаток списка
        next := curr.Next
        
        // 2. Разворачиваем указатель текущего элемента назад
        curr.Next = prev
        
        // 3. Сдвигаем 'окно' на один шаг вперед
        prev = curr
        curr = next
    }
    
    // В конце curr укажет на nil, а prev — на новую голову списка
    return prev
}

3. Паттерн "Быстрый и Медленный указатель" (Fast & Slow Pointers)​

Этот паттерн (также известный как Алгоритм Флойда или Черепаха и Заяц) используется в связанных списках для поиска циклов или середины списка.

Задача: Поиск середины списка​

Медленный указатель шагает по 1 узлу, быстрый — по 2. Когда быстрый доходит до конца, медленный оказывается ровно на середине.

func middleNode(head *ListNode) *ListNode {
    slow := head
    fast := head
    
    // Пока быстрый указатель может сделать 2 шага вперед
    for fast != nil && fast.Next != nil {
        slow = slow.Next       // Шаг на 1
        fast = fast.Next.Next  // Шаг на 2
    }
    
    return slow
}

Задача: Обнаружение цикла (Cycle Detection)​

Если в связанном списке есть цикл (какой-то узел ссылается на предыдущий), то "быстрый заяц", бегущий по циклу, неминуемо догонит "медленную черепаху". Если цикла нет, быстрый просто добежит до конца (nil).

func hasCycle(head *ListNode) bool {
    if head == nil {
        return false
    }
    
    slow := head
    fast := head
    
    for fast != nil && fast.Next != nil {
        slow = slow.Next
        fast = fast.Next.Next
        
        // Заяц догнал черепаху — мы бегаем по кругу!
        if slow == fast {
            return true
        }
    }
    
    return false
}

4. Слияние двух отсортированных списков​

Ещё одна классика (LeetCode #21). Даны головы двух отсортированных списков, нужно слить их в один отсортированный.

Используется "фиктивная голова" (dummy node), чтобы не писать сложную логику для первого элемента.

func mergeTwoLists(list1 *ListNode, list2 *ListNode) *ListNode {
    dummy := &ListNode{} // Фиктивный узел
    current := dummy     // Указатель для сборки нового списка
    
    for list1 != nil && list2 != nil {
        if list1.Val < list2.Val {
            current.Next = list1
            list1 = list1.Next
        } else {
            current.Next = list2
            list2 = list2.Next
        }
        current = current.Next // Двигаем указатель сборщика
    }
    
    // Добавляем остатки (если один список закончился раньше другого)
    if list1 != nil {
        current.Next = list1
    } else {
        current.Next = list2
    }
    
    return dummy.Next // Возвращаем всё, что идет после фиктивного узла
}
