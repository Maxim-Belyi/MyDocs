Деревья (Trees)

Дерево — это иерархическая структура данных, состоящая из узлов. Каждый узел имеет значение и ссылки на дочерние узлы (потомков).

Самый популярный вид деревьев на собеседованиях — Бинарное дерево (Binary Tree), где каждый узел может иметь не более двух потомков (левого и правого).

1. Структура бинарного дерева в Go​

type TreeNode struct {
    Val   int
    Left  *TreeNode
    Right *TreeNode
}

2. Обходы дерева (Tree Traversals)​

Чаще всего на собеседованиях просят обойти дерево. Существует два основных подхода: в глубину (DFS) и в ширину (BFS).

Обход в глубину (DFS - Depth-First Search)​

Обычно реализуется рекурсивно. Идет по ветке максимально глубоко, прежде чем вернуться назад. Различают три порядка обхода:

  1. Pre-order (Прямой): Узел -> Левое поддерево -> Правое поддерево
  2. In-order (Симметричный): Левое поддерево -> Узел -> Правое поддерево (если применить к бинарному дереву поиска (BST), вернет узлы в отсортированном порядке).
  3. Post-order (Обратный): Левое поддерево -> Правое поддерево -> Узел (удобно для удаления дерева или расчета высоты).

Пример In-order обхода (сборка всех значений дерева в срез):

func inorderTraversal(root *TreeNode) []int {
    var result []int
    
    // Вспомогательная рекурсивная функция
    var traverse func(node *TreeNode)
    traverse = func(node *TreeNode) {
        if node == nil {
            return
        }
        traverse(node.Left)        // Сначала идем максимально влево
        result = append(result, node.Val) // Читаем значение
        traverse(node.Right)       // Затем идем вправо
    }
    
    traverse(root)
    return result
}

Обход в ширину (BFS - Breadth-First Search)​

Обходит дерево по "уровням". Сначала корень, потом все дети корня, потом все внуки и т.д. Реализуется итеративно с использованием очереди.

func levelOrder(root *TreeNode) [][]int {
    if root == nil {
        return nil
    }
    
    var result [][]int
    queue := []*TreeNode{root} // Очередь (slice) с корневым узлом
    
    for len(queue) > 0 {
        levelSize := len(queue) // Сколько элементов на текущем уровне
        var currentLevel []int
        
        // Обрабатываем только узлы текущего уровня
        for i := 0; i < levelSize; i++ {
            // Извлекаем первый элемент из очереди (pop)
            node := queue[0]
            queue = queue[1:]
            
            currentLevel = append(currentLevel, node.Val)
            
            // Добавляем детей в конец очереди
            if node.Left != nil {
                queue = append(queue, node.Left)
            }
            if node.Right != nil {
                queue = append(queue, node.Right)
            }
        }
        
        result = append(result, currentLevel)
    }
    
    return result
}

3. Бинарное Дерево Поиска (Binary Search Tree - BST)​

BST — это особое бинарное дерево со строгим правилом: для любого узла все значения в левом поддереве меньше значения узла, а в правом — больше. Это позволяет находить, добавлять и удалять элементы за $O(\log N)$ времени.

Классическая задача: Валидация BST​

Как проверить, что данное дерево является правильным BST? Недостаточно просто проверить Left < Node < Right. Нужно проверять диапазон для всего поддерева.

func isValidBST(root *TreeNode) bool {
    var validate func(node *TreeNode, min, max *int) bool
    validate = func(node *TreeNode, min, max *int) bool {
        if node == nil {
            return true
        }
        
        // Проверяем нарушение границ
        if min != nil && node.Val <= *min {
            return false
        }
        if max != nil && node.Val >= *max {
            return false
        }
        
        // Левое поддерево должно быть меньше текущего узла (max = node.Val)
        // Правое поддерево должно быть больше текущего узла (min = node.Val)
        return validate(node.Left, min, &node.Val) && validate(node.Right, &node.Val, max)
    }
    
    return validate(root, nil, nil)
}
