# Бинарный поиск (Binary Search)

Бинарный поиск — это классический алгоритм поиска элемента в **отсортированном** массиве (или другой монотонной структуре). 

Вместо того чтобы проверять каждый элемент (что даёт сложность O(N)), бинарный поиск делит массив пополам на каждом шаге, сравнивая искомое значение со средним элементом. Это дает логарифмическую сложность **O(log N)**, что невероятно быстро: среди 1 миллиона элементов алгоритм найдет ответ максимум за 20 шагов.

---

## 1. Базовая реализация "руками"

Классический алгоритм пишется через индексы `left` и `right`.

```go
func binarySearch(nums []int, target int) int {
    left := 0
    right := len(nums) - 1
    
    for left <= right {
        // Избегаем переполнения, хотя в Go int обычно 64-битный и переполнение маловероятно.
        // Эквивалент (left + right) / 2
        mid := left + (right - left) / 2 
        
        if nums[mid] == target {
            return mid     // Элемент найден
        } else if nums[mid] < target {
            left = mid + 1 // Искомое значение больше, ищем в правой половине
        } else {
            right = mid - 1 // Искомое значение меньше, ищем в левой половине
        }
    }
    
    return -1 // Элемент не найден
}
```

> **Важно:** Всегда помните условие `left <= right` (со знаком равно), иначе алгоритм может не проверить последний оставшийся элемент.

## 2. Бинарный поиск по предикату

Обычно бинарный поиск ищет не просто "значение в массиве", а "первое значение, удовлетворяющее условию". Например, "найти первый коммит, в котором сломалась сборка" (когда все до него были успешны, а все после — сломаны).

Это называется **монотонная функция** (например, `[false, false, false, true, true, true]`). Мы хотим найти индекс первой `true`.

```go
// Ищем первый элемент, который >= target
func searchInsertPosition(nums []int, target int) int {
    left := 0
    right := len(nums) - 1
    ans := len(nums) // По умолчанию элемент больше всех
    
    for left <= right {
        mid := left + (right - left) / 2
        
        if nums[mid] >= target {
            ans = mid      // Запоминаем возможный ответ
            right = mid - 1 // Продолжаем искать ещё левее
        } else {
            left = mid + 1  // Искомое условие не выполнено, идем вправо
        }
    }
    
    return ans
}
```

## 3. Встроенные средства Go

В стандартной библиотеке Go уже есть отличные инструменты для бинарного поиска.

### Пакет `sort` (Универсальный поиск)
Функция `sort.Search` реализует именно поиск по предикату. Она принимает размер и функцию, возвращающую `bool`. Она ищет наименьший индекс `i`, для которого `f(i)` возвращает `true`.

```go
import "sort"

func main() {
    nums := []int{1, 3, 5, 7, 9}
    target := 6
    
    // Найдем первый индекс, где элемент >= 6
    i := sort.Search(len(nums), func(i int) bool {
        return nums[i] >= target
    })
    
    // i будет равно 3 (индекс числа 7)
    fmt.Println("Insert position:", i)
}
```

### Пакет `slices` (С Go 1.21+)
В новых версиях Go появился обобщенный (generic) бинарный поиск, который гораздо удобнее для стандартных срезов:

```go
import "slices"

func main() {
    nums := []int{10, 20, 30, 40, 50}
    
    // Ищет точное совпадение. 
    // Возвращает индекс и флаг: найден (true) или нет (false)
    idx, found := slices.BinarySearch(nums, 30)
    fmt.Printf("Index: %d, Found: %v\n", idx, found) // Index: 2, Found: true
    
    // Если элемента нет, idx указывает, куда его вставить, чтобы сохранить сортировку
    idx, found = slices.BinarySearch(nums, 25)
    fmt.Printf("Index: %d, Found: %v\n", idx, found) // Index: 2, Found: false
}
```

## Итог
- Если данные **уже отсортированы**, бинарный поиск — лучший инструмент.
- Если нужно написать руками, используйте шаблон `left = 0, right = len - 1`, цикл `while left <= right` и `mid = left + (right - left) / 2`.
- В рабочем коде используйте `slices.BinarySearch` или `sort.Search`.
