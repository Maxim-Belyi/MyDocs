Хэш-таблицы (Hash Tables)

Хэш-таблица — одна из важнейших структур данных в Computer Science, обеспечивающая вставку, поиск и удаление элементов за амортизированное время O(1).

В языке Go хэш-таблица реализована встроенным типом map.

1. Базовые принципы​

Ключевая идея хэш-таблицы заключается в использовании хэш-функции, которая принимает ключ (например, строку "user_id") и возвращает целое число (индекс в массиве), по которому будут храниться данные.

Коллизии​

Идеальных хэш-функций не существует, поэтому разные ключи могут дать одинаковый индекс. Это называется коллизией. В Go коллизии разрешаются с помощью метода цепочек (chaining) в комбинации с открытой адресацией (внутри "бакетов"). Под капотом map в Go состоит из массива бакетов (buckets), каждый из которых хранит до 8 пар ключ-значение. Если бакет переполняется, создается дополнительный бакет, связанный с ним (overflow bucket).

2. Использование map в Go​

Инициализация​

Всегда инициализируйте map перед записью! Запись в неинициализированную (nil) мапу вызовет панику.

// Правильно:
m := make(map[string]int) 
// Если заранее известен примерный размер, лучше указать его для оптимизации:
// m := make(map[string]int, 100)

// Паника:
// var m map[string]int
// m["key"] = 1 // panic: assignment to entry in nil map

Проверка наличия ключа​

При чтении несуществующего ключа Go возвращает zero-value для типа значения. Чтобы отличить отсутствие ключа от реального нуля, используется идиома "comma ok":

val, exists := m["alice"]
if !exists {
    fmt.Println("Пользователь не найден")
}

Хэш-сеты (Set)​

В Go нет встроенного типа Set (множество уникальных элементов). Его эмулируют через map[T]struct{}. Пустая структура struct{} не занимает памяти (0 байт), что делает такой Set максимально эффективным.

visited := make(map[string]struct{})

// Добавление
visited["/home"] = struct{}{}

// Проверка
if _, ok := visited["/home"]; ok {
    fmt.Println("Страница уже посещалась")
}

3. На собеседовании​

Популярные вопросы:

  1. Какова сложность операций в map? В среднем O(1), но в худшем случае (когда все ключи попадают в один бакет) деградирует до O(N).
  2. Является ли map потокобезопасной? НЕТ. Конкурентная запись и чтение приведут к fatal error: concurrent map read and map write. Для потокобезопасности нужно использовать sync.RWMutex или sync.Map.
  3. Какие типы могут быть ключами? Только сравнимые типы (comparable). Это числа, строки, булевы значения, указатели, интерфейсы, каналы, массивы и структуры, состоящие из сравнимых типов. Срезы (slices), мапы и функции не могут быть ключами.

Классическая задача: Two Sum​

Часто встречается на LeetCode (уровень Easy). Задача: Найти в массиве два числа, сумма которых равна target, и вернуть их индексы.

Решение через Хэш-таблицу (O(N) по времени и памяти): Вместо вложенных циклов мы один раз проходим по массиву, сохраняя пройденные числа в map в формате число: индекс. Для каждого числа мы ищем его "пару" (target - текущее_число) в мапе.

func twoSum(nums []int, target int) []int {
    // Ключ: само число, Значение: его индекс в массиве
    seen := make(map[int]int, len(nums))
    
    for i, num := range nums {
        complement := target - num // Ищем "дополнение"
        
        if j, found := seen[complement]; found {
            return []int{j, i} // Нашли пару!
        }
        
        seen[num] = i // Сохраняем число для будущих проверок
    }
    
    return nil // Пара не найдена
}
