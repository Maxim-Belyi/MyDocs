Графы (Graphs)

Граф — это структура данных, состоящая из вершин (узлов) и ребер, которые их соединяют. Графы могут быть ориентированными (связи имеют направление) и неориентированными, взвешенными (ребро имеет "стоимость") и невзвешенными.

Деревья и связанные списки — это частные случаи графов. С помощью графов моделируют дорожные сети, социальные связи (друзья в VK), компьютерные сети, зависимости между пакетами кода.

1. Как представить граф в Go​

Чаще всего графы реализуют через Список смежности (Adjacency List). В Go это удобно делать с помощью map, где ключ — это вершина, а значение — список (срез) соседних вершин.

// Представление неориентированного графа
type Graph map[string][]string

func main() {
    g := make(Graph)
    
    // Добавление ребер
    // A --- B
    // |     |
    // C --- D
    g["A"] = []string{"B", "C"}
    g["B"] = []string{"A", "D"}
    g["C"] = []string{"A", "D"}
    g["D"] = []string{"B", "C"}
}

2. Обход графа​

Как и в деревьях, для графов существуют два основных метода обхода: DFS (в глубину) и BFS (в ширину). Главное отличие от деревьев: в графах могут быть циклы! Поэтому нам обязательно нужен хэш-сет visited, чтобы не застрять в бесконечном цикле.

Обход в глубину (DFS)​

Используется для поиска пути, обнаружения циклов, топологической сортировки.

func dfs(g Graph, start string) {
    visited := make(map[string]bool)
    
    var traverse func(node string)
    traverse = func(node string) {
        if visited[node] {
            return // Уже были здесь, предотвращаем цикл
        }
        
        visited[node] = true
        fmt.Println("Посетили:", node)
        
        // Рекурсивно идем к соседям
        for _, neighbor := range g[node] {
            traverse(neighbor)
        }
    }
    
    traverse(start)
}

Обход в ширину (BFS)​

Полезен для поиска кратчайшего пути (с минимальным количеством шагов) в невзвешенном графе. Реализуется через очередь (Queue).

func bfs(g Graph, start string) {
    visited := make(map[string]bool)
    queue := []string{start} // Наша очередь
    
    visited[start] = true
    
    for len(queue) > 0 {
        // Достаем из начала очереди
        node := queue[0]
        queue = queue[1:]
        
        fmt.Println("Посетили:", node)
        
        // Добавляем всех соседей в конец очереди
        for _, neighbor := range g[node] {
            if !visited[neighbor] {
                visited[neighbor] = true // Отмечаем сразу при добавлении в очередь!
                queue = append(queue, neighbor)
            }
        }
    }
}

3. Топологическая сортировка​

Частая алгоритмическая задача на графы. Даны курсы в университете и их зависимости (какой курс нужно пройти до какого). Нужно определить порядок, в котором можно пройти все курсы.

Это задача топологической сортировки ориентированного ациклического графа (DAG). Решается модифицированным DFS.

// Задача решается с помощью массива состояний узлов:
// 0 - не посещен, 1 - в процессе посещения (поиск цикла), 2 - полностью обработан

func topologicalSort(numCourses int, prerequisites [][]int) []int {
    // 1. Строим граф
    graph := make([][]int, numCourses)
    for _, req := range prerequisites {
        course, pre := req[0], req[1]
        graph[pre] = append(graph[pre], course)
    }
    
    states := make([]int, numCourses)
    var result []int
    var hasCycle bool
    
    var dfs func(node int)
    dfs = func(node int) {
        if states[node] == 1 {
            hasCycle = true // Нашли цикл, курсы пройти невозможно!
            return
        }
        if states[node] == 2 || hasCycle {
            return
        }
        
        states[node] = 1 // Отмечаем "в процессе"
        for _, neighbor := range graph[node] {
            dfs(neighbor)
        }
        
        states[node] = 2 // Отмечаем "обработан"
        // Добавляем в результат в ОБРАТНОМ порядке завершения
        // В Go проще добавить в конец, а потом развернуть весь массив
        result = append(result, node) 
    }
    
    for i := 0; i < numCourses; i++ {
        if states[i] == 0 {
            dfs(i)
        }
    }
    
    if hasCycle {
        return nil
    }
    
    // Разворачиваем результат для топологической сортировки
    for i, j := 0, len(result)-1; i < j; i, j = i+1, j-1 {
        result[i], result[j] = result[j], result[i]
    }
    
    return result
}
