# Дженерики (Generics) в Go

До версии 1.18 в Go отсутствовала поддержка обобщенного программирования. Если нужно было написать функцию для поиска элемента в массиве, приходилось либо писать отдельную функцию для каждого типа (`int`, `string`, `float64`), либо использовать `interface{}` и `reflect`, что было медленно и нетипобезопасно.

**Дженерики (Generics)** решают эту проблему, позволяя писать функции и структуры данных, которые работают с любым типом, сохраняя при этом строгую типизацию во время компиляции.

---

## 1. Базовый синтаксис: Функции с параметрами типа

Дженерик-функции объявляются с помощью квадратных скобок `[]` перед обычными параметрами функции.

```go
// T - это параметр типа. `any` - это ограничение (constraint), означающее "любой тип".
func PrintAnything[T any](thing T) {
    fmt.Println(thing)
}

func main() {
    PrintAnything[int](42)        // Явное указание типа
    PrintAnything("Hello")        // Неявное выведение типа компилятором
    PrintAnything(3.14)
}
```

Ключевое слово `any` — это просто алиас для `interface{}`, добавленный в Go 1.18.

---

## 2. Ограничения типов (Constraints)

Часто функциям нужно не просто "что-то", а что-то, что можно, например, сравнивать (`==`, `!=`) или складывать (`+`). Для этого используются ограничения.

### Встроенное ограничение `comparable`
Позволяет использовать операторы `==` и `!=`.

```go
// T должен быть сравним (числа, строки, булевы значения и т.д.)
func Contains[T comparable](slice []T, target T) bool {
    for _, item := range slice {
        if item == target {
            return true
        }
    }
    return false
}

func main() {
    numbers := []int{1, 2, 3, 4, 5}
    fmt.Println(Contains(numbers, 3)) // true
}
```

### Кастомные ограничения (Интерфейсы с Union Types)
Вы можете создать свой интерфейс, перечислив в нём разрешенные типы через оператор `|`.

```go
// Ограничение: T может быть только int или float64
type Number interface {
    int | float64
}

func Sum[T Number](a, b T) T {
    return a + b // Оператор '+' работает, так как мы ограничили T числами
}

func main() {
    fmt.Println(Sum(5, 10))       // 15
    fmt.Println(Sum(3.14, 2.0))   // 5.14
    // Sum("a", "b") // Ошибка компиляции: string не удовлетворяет интерфейсу Number
}
```

## 3. Обобщенные структуры (Generic Structs)

Дженерики применяются не только к функциям, но и к структурами данных. Это идеально подходит для создания собственных стеков, очередей, списков или деревьев.

```go
// Стек, который может хранить элементы любого типа
type Stack[T any] struct {
    elements []T
}

func (s *Stack[T]) Push(value T) {
    s.elements = append(s.elements, value)
}

func (s *Stack[T]) Pop() (T, bool) {
    if len(s.elements) == 0 {
        var zero T // Возвращаем нулевое значение типа T (например 0 для int, "" для string)
        return zero, false
    }
    
    index := len(s.elements) - 1
    value := s.elements[index]
    s.elements = s.elements[:index]
    return value, true
}

func main() {
    intStack := Stack[int]{}
    intStack.Push(10)
    intStack.Push(20)
    
    val, _ := intStack.Pop()
    fmt.Println(val) // 20
}
```

> **Важно:** Обрати внимание на `var zero T`. Так как мы заранее не знаем тип, мы объявляем пустую переменную типа `T` и Go автоматически инициализирует её нулевым значением (zero-value) для этого типа.

## 4. Использование `~` (Тильда) в ограничениях

Иногда у нас есть собственные типы, основанные на базовых:
```go
type MyInt int
```
Если мы используем ограничение `Number interface { int | float64 }`, то `MyInt` не подойдет. Чтобы разрешить все типы, *в основе которых лежит `int`*, используется оператор `~`.

```go
type FlexibleNumber interface {
    ~int | ~float64
}

func PrintDouble[T FlexibleNumber](val T) {
    fmt.Println(val * 2)
}

func main() {
    var myVal MyInt = 10
    PrintDouble(myVal) // Работает! Выведет 20
}
```

## 5. Стандартные пакеты (slices, maps, cmp)

Вместе с дженериками в Go добавили полезные встроенные пакеты для работы с коллекциями:

*   **`slices`**: Функции `slices.Contains`, `slices.Sort`, `slices.Insert`, `slices.Delete`, `slices.Reverse` и другие. Больше не нужно писать свою логику сортировки для слайса интов!
*   **`maps`**: Функции `maps.Keys`, `maps.Values`, `maps.Clone`, `maps.Copy`.
*   **`cmp`**: Содержит пакеты `cmp.Ordered` (интерфейс для типов, поддерживающих операторы `<, <=, >, >=`) и `cmp.Compare()`.

Пример использования:
```go
import "slices"

func main() {
    words := []string{"apple", "banana", "cherry"}
    slices.Reverse(words)
    fmt.Println(words) // [cherry banana apple]
}
```

## Когда НЕ нужно использовать дженерики?

Создатели Go (Роб Пайк, Иэн Тейлор) часто предупреждают: не злоупотребляйте дженериками.
1.  **Если интерфейс делает работу лучше.** Если метод просто должен удовлетворять определенному поведению (например `io.Reader`), используйте классические интерфейсы.
2.  **Не усложняйте код.** Дженерики делают код тяжелее для чтения. Если функция используется только для `int`, не нужно делать её дженериком "на будущее". 

> **Золотое правило:** Пишите код, работающий с конкретными типами. Вводите параметры типа (Generics) только тогда, когда замечаете, что пишете абсолютно одинаковый бойлерплейт для разных типов данных.
