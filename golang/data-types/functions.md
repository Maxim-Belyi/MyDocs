# Функции в Go
В Go функции — это основной строительный блок приложения. Здесь нет классов и конструкторов, поэтому вся логика живет в функциях и методах.

## 1. Множественный возврат значений
Go — один из языков, где функция может возвращать сразу несколько значений "из коробки". Чаще всего это используется для идиоматичной обработки ошибок (`result, err`).

```go
func divide(a, b float64) (float64, error) {
    if b == 0 {
        return 0, errors.New("деление на ноль")
    }
    return a / b, nil
}
```

### Именованные возвращаемые значения (Named Return Values)
При объявлении функции можно заранее дать имена возвращаемым переменным.
*   Они создаются автоматически и инициализируются "нулевыми значениями" (zero-values).
*   Позволяют использовать **«голый возврат» (naked return)** — вызов `return` без указания переменных.

```go
func split(sum int) (x, y int) {
    x = sum * 4 / 9
    y = sum - x
    return // "Голый возврат" вернет текущие значения x и y
}
```
> **Best Practice:** Именованные возвращаемые значения отлично подходят для документирования кода (сразу понятно, что функция возвращает). Но **голый возврат** в длинных функциях использовать *не рекомендуется*, так как он ухудшает читаемость — разработчику сложно понять, что именно возвращается, не пролистывая код наверх.

## 2. Вариативные функции (Variadic functions)
Функция может принимать неограниченное количество аргументов одного типа. Для этого используется синтаксис `...Type`. Внутри функции такие аргументы превращаются в обычный слайс (slice).

```go
func sum(nums ...int) int {
    total := 0
    for _, n := range nums { // nums здесь - это[]int
        total += n
    }
    return total
}

func main() {
    fmt.Println(sum(1, 2))       // Передача аргументов через запятую
    fmt.Println(sum(1, 2, 3, 4)) 

    // Если у нас уже есть слайс, мы можем его "распаковать" при передаче:
    mySlice :=[]int{10, 20, 30}
    fmt.Println(sum(mySlice...)) // Распаковка слайса!
}
```

## 3. Функции первого класса (First-class functions)
В Go функцию можно присвоить переменной, передать как аргумент или вернуть из другой функции. Это основа паттерна "Callback" и функционального программирования.

```go
// Объявляем тип для функции (чтобы не писать длинную сигнатуру каждый раз)
type mathOp func(int, int) int

// Функция принимает другую функцию как аргумент
func calculate(a, b int, op mathOp) int {
    return op(a, b)
}

func main() {
    add := func(x, y int) int { return x + y }
    multiply := func(x, y int) int { return x * y }

    fmt.Println(calculate(5, 3, add))      // 8
    fmt.Println(calculate(5, 3, multiply)) // 15
}
```

## 4. Анонимные функции и Замыкания (Closures) 

Анонимная функция — это функция без имени, объявленная прямо внутри другой функции. 

**Замыкание (Closure)** — это анонимная функция, которая "захватывает" (запоминает) переменные из внешнего контекста, в котором она была создана. Даже когда внешняя функция завершает работу, замыкание продолжает иметь доступ к этим переменным.

```go
// Функция возвращает другую функцию
func counter() func() int {
    count := 0 // Локальная переменная
    
    // Возвращаем анонимную функцию (замыкание)
    return func() int {
        count++ // Она ЗАХВАТИЛА переменную count
        return count
    }
}

func main() {
    c1 := counter()
    fmt.Println(c1()) // 1
    fmt.Println(c1()) // 2

    c2 := counter()   // Создается новый, независимый контекст
    fmt.Println(c2()) // 1
}
```

### ⚠️ Ловушка замыканий в цикле 

На интервью любят давать вот такой код и спрашивать: *"Что выведется на экран?"*

```go
func main() {
    for i := 0; i < 3; i++ {
        // Создаем анонимную горутину (замыкание)
        go func() {
            fmt.Println(i)
        }()
    }
    
    time.Sleep(time.Second) // Ждем завершения горутин
}
```

**Ответ (до версии Go 1.22):** Выведется `3 3 3`.
**Почему?** Анонимная функция захватывает переменную `i` **по ссылке**, а не по значению. Цикл `for` работает молниеносно, и к моменту, когда горутины реально запускаются планировщиком операционной системы, цикл уже завершился, и переменная `i` равна `3`. Все горутины смотрят в один и тот же адрес памяти.

**Как это чинили раньше:**
Передавали значение как аргумент (тогда создается локальная копия):
```go
go func(val int) {
    fmt.Println(val)
}(i) // передаем текущее значение i
```

🔥 **Козырь для собеседования:** Обязательно упомяни, что **начиная с версии Go 1.22 это поведение исправили на уровне языка!** Теперь цикл `for` на каждой итерации создает *новую* переменную `i`. Поэтому в современных версиях Go этот код выведет `0 1 2` (в случайном порядке). Интервьюеры обожают, когда кандидат следит за обновлениями языка.

## 5. Отложенные вызовы (`defer`)

Ключевое слово `defer` позволяет "отложить" выполнение функции до момента, когда функция-родитель (в которой вызван `defer`) завершит свою работу (через `return` или из-за `panic`).

Чаще всего `defer` используется для освобождения ресурсов, чтобы не забыть это сделать:
*   Закрытие файлов (`file.Close()`).
*   Снятие блокировок мьютексов (`mu.Unlock()`).
*   Закрытие соединений с БД или HTTP-телом (`resp.Body.Close()`).

### Три главных правила `defer`:

#### Правило 1: Аргументы вычисляются немедленно
Аргументы для отложенной функции вычисляются в момент *объявления* `defer`, а не в момент её *выполнения*.

```go
func main() {
    name := "Alex"
    defer fmt.Println("Пока,", name) // Запомнит "Alex" прямо сейчас
    
    name = "Ivan"
    // Выведет: "Пока, Alex"
}
```

#### Правило 2: Порядок выполнения LIFO (Last In, First Out)
Если в функции несколько вызовов `defer`, они помещаются в стек и выполняются в обратном порядке: "Последний зашел — первый вышел".

```go
func main() {
    defer fmt.Println("1")
    defer fmt.Println("2")
    defer fmt.Println("3")
    // Выведет: 3, 2, 1
}
```

#### Правило 3: `defer` может изменять именованные возвращаемые значения
Это супер-важный трюк. Если у функции есть **именованное** возвращаемое значение, `defer` имеет к нему доступ и может изменить его *после* того, как отработал `return`!

```go
func trickyReturn() (result int) {
    defer func() {
        result++ // Увеличиваем результат перед самым выходом
    }()
    
    return 10 // Сначала result = 10, потом срабатывает defer, и возвращается 11!
}
```
*(Именно этот механизм используется в связке с `recover` для перехвата паник и возвращения красивых ошибок).*
