Функции как объекты первого класса (First-Class Functions)

В языке Go функции являются "объектами первого класса" (First-Class Citizens). Это означает, что функции обладают теми же правами, что и обычные переменные:

  • Их можно присваивать переменным.
  • Их можно передавать в качестве аргументов в другие функции (создавая функции высшего порядка — Higher-Order Functions).
  • Их можно возвращать из других функций.
  • Их можно хранить в структурах данных (массивах, мапах, полях структур).

Этот паттерн позволяет писать очень гибкий, расширяемый и тестируемый код, избегая создания громоздких интерфейсов там, где достаточно передать одно простое действие (поведение).

1. Объявление пользовательских типов функций​

Хотя в коде можно использовать сырую сигнатуру (например, func(ctx context.Context) error), в Go принято создавать для них отдельные типы. Это делает код намного читаемее и позволяет привязывать к таким функциям дополнительные методы.

// Объявляем тип Operation, который является функцией
type Operation func(ctx context.Context) error

Теперь мы можем использовать Operation везде, где ожидается функция с такой сигнатурой.

2. Практический пример: Менеджер Graceful Shutdown​

Ваш каркас — это отличный пример использования этого паттерна для создания переиспользуемого менеджера плавной остановки сервиса (Graceful Shutdown). Вместо того чтобы жестко "зашивать" в менеджер знания о том, как закрывать базу данных, кэш или брокер сообщений, мы можем позволить другим модулям просто регистрировать свои функции "очистки".

Вот как выглядит законченная реализация этого паттерна:

package shutdown

import (
    "context"
    "fmt"
    "sync"
    "time"
)

// Operation описывает функцию завершения работы
type Operation func(ctx context.Context) error

// GracefulShutdown хранит набор функций-операций для закрытия ресурсов
type GracefulShutdown struct {
    operations map[string]Operation
    mu         sync.Mutex // Защищает map от конкурентной записи (полезно, если Add вызывают из разных горутин)
}

// New создает новый экземпляр менеджера
func New() *GracefulShutdown {
    return &GracefulShutdown{
        operations: make(map[string]Operation),
    }
}

// Add регистрирует новую операцию (функцию) в менеджере
func (gs *GracefulShutdown) Add(name string, op Operation) {
    gs.mu.Lock()
    defer gs.mu.Unlock()
    gs.operations[name] = op
}

Реализация метода Wait​

Теперь, когда функции хранятся в нашей структуре GracefulShutdown, мы можем перебрать их и вызвать. Поскольку операции закрытия независимы, их можно выполнять параллельно через горутины, контролируя общий таймаут через context.

// Wait выполняет все зарегистрированные функции с общим таймаутом
func (gs *GracefulShutdown) Wait(timeout time.Duration) {
    ctx, cancel := context.WithTimeout(context.Background(), timeout)
    defer cancel()

    var wg sync.WaitGroup
    
    // Безопасно копируем карту операций, чтобы не блокировать мьютекс на время их выполнения
    gs.mu.Lock()
    ops := gs.operations
    gs.mu.Unlock()

    for name, op := range ops {
        wg.Add(1)
        
        // Запускаем каждую операцию (функцию) параллельно
        go func(opName string, operation Operation) {
            defer wg.Done()
            
            // Вызываем сохраненную функцию как обычный метод
            if err := operation(ctx); err != nil {
                fmt.Printf("Ошибка при завершении %s: %v\n", opName, err)
            } else {
                fmt.Printf("Операция %s успешно завершена\n", opName)
            }
        }(name, op)
    }

    // Ожидаем завершения всех горутин
    done := make(chan struct{})
    go func() {
        wg.Wait()
        close(done)
    }()

    select {
    case <-done:
        fmt.Println("Все сервисы остановлены штатно")
    case <-ctx.Done():
        fmt.Println("Принудительная остановка по таймауту")
    }
}

Как это используется вызывающим кодом?​

Магия функций первого класса раскрывается здесь — за счет замыканий (closures). Функция "захватывает" переменные из внешней области видимости (например, db), поэтому менеджеру GracefulShutdown не нужно ничего знать про тип БД.

func main() {
    gs := shutdown.New()

    // db инициализирована где-то ранее
    // База данных регистрирует свою функцию остановки
    gs.Add("Database", func(ctx context.Context) error {
        // Мы имеем доступ к db благодаря замыканию!
        return db.Close() 
    })

    // Брокер сообщений регистрирует свою функцию
    gs.Add("RabbitMQ", func(ctx context.Context) error {
        return rabbitmq.Close()
    })

    // Ждем 5 секунд на завершение всех операций
    gs.Wait(5 * time.Second)
}

3. Другие примеры использования паттерна​

Middleware в HTTP-серверах​

Любой HTTP-роутер (например, стандартный net/http или chi) основан на этом принципе. Middleware — это функция высшего порядка, которая принимает одну функцию (следующий обработчик) и возвращает другую (обертку).

func LoggerMiddleware(next http.HandlerFunc) http.HandlerFunc {
    // Возвращаем новую функцию, которая соответствует сигнатуре HandlerFunc
    return func(w http.ResponseWriter, r *http.Request) {
        start := time.Now()
        
        // Вызов переданной функции (оригинального обработчика)
        next(w, r) 
        
        fmt.Printf("Запрос занял %v\n", time.Since(start))
    }
}

Паттерн "Функциональные опции" (Functional Options)​

Идеально подходит для создания структур с множеством опциональных параметров без необходимости плодить десятки функций-конструкторов (как это делают в Java/C# с помощью перегрузки или паттерна Builder).

type Server struct {
    host string
    port int
}

// Option - это функция, которая меняет состояние сервера
type Option func(*Server)

// Функция возвращает функцию!
func WithPort(port int) Option {
    return func(s *Server) {
        s.port = port
    }
}

func NewServer(opts ...Option) *Server {
    srv := &Server{host: "localhost", port: 8080} // значения по умолчанию
    
    // Применяем все переданные функции-опции к серверу
    for _, opt := range opts {
        opt(srv)
    }
    
    return srv
}

// Использование в коде:
server := NewServer(WithPort(3000))

Итог: Чек-лист​

  1. Типизация: если сигнатура функции сложная (больше 1 аргумента или она возвращает значения), всегда выносите её в отдельный type (например, type Operation func(...)).
  2. Изоляция логики (Dependency Injection): передача функции — это идеальный способ внедрения зависимостей, когда вам не нужен полноценный interface со множеством методов. Если нужно выполнить ровно одно действие, просите функцию.
  3. Замыкания (Closures): передаваемые анонимные функции часто являются замыканиями (как в main в примере с db.Close()), что позволяет им использовать переменные из внешней области видимости, не требуя их явной передачи через аргументы.
