# Log

Пакет `log` предоставляет простые инструменты для форматированного логирования. Он выводит сообщения со стандартным форматированием, которое включает дату и время записи.

---

## 1. Базовый вывод: Print, Println, Printf

По умолчанию логи выводятся в стандартный поток ошибок (`os.Stderr`).

```go
package main

import "log"

func main() {
    log.Print("Запуск сервиса...")
    log.Println("Подключение к БД установлено")
    log.Printf("Обработка запроса от пользователя ID: %d", 42)
}
```
**Результат в консоли:**
`2024/05/20 15:04:05 Подключение к БД установлено`

---

## 2. Логирование с завершением: Fatal и Panic

Эти функции используются при критических ошибках, когда продолжение работы программы невозможно или небезопасно.

### log.Fatal
Выводит сообщение и завершает программу с кодом ошибки (вызывает `os.Exit(1)`). При этом отложенные вызовы (`defer`) **не выполняются**.

```go
file, err := os.Open("config.json")
if err != nil {
    log.Fatalf("ошибка загрузки конфигурации: %v", err)
}
```

### log.Panic
Выводит сообщение и вызывает панику (`panic()`). В отличие от `Fatal`, позволяет отложенным вызовам (`defer`) отработать перед аварийным завершением.

```go
if db == nil {
    log.Panic("соединение с базой данных не инициализировано")
}
```

## 3. Настройка логгера: SetFlags и SetPrefix

Вы можете изменить формат вывода стандартного логгера.

*   `SetPrefix`: добавляет фиксированную строку перед каждым сообщением.
*   `SetFlags`: позволяет добавить вывод имени файла, номера строки или изменить формат времени.

```go
package main

import "log"

func init() {
    log.SetPrefix("[API] ")
    // Ldate и Ltime - стандартные. Lshortfile добавляет имя файла и строку.
    log.SetFlags(log.Ldate | log.Ltime | log.Lshortfile)
}

func main() {
    log.Println("Сервер слушает порт 8080")
}
```
**Результат:**
`[API] 2024/05/20 15:04:05 main.go:12: Сервер слушает порт 8080`

## 4. Создание пользовательских логгеров: log.New

Если вам нужно писать логи в разные места (например, в консоль и в файл одновременно, или разделить лог ошибок и информационный лог), создайте свой экземпляр логгера.

```go
package main

import (
    "log"
    "os"
)

func main() {
    file, err := os.OpenFile("app.log", os.O_CREATE|os.O_WRONLY|os.O_APPEND, 0666)
    if err != nil {
        log.Fatal("Не удалось открыть файл логов")
    }
    defer file.Close()

    // Создаем логгер, который пишет в файл
    infoLog := log.New(file, "INFO: ", log.Ldate|log.Ltime|log.Lshortfile)
    
    infoLog.Println("Логгер успешно инициализирован")
}
```

---

## Итог: Чек-лист по пакету log

1.  **Простота:** используйте `log.Println` или `log.Printf` для базовых информационных сообщений и ошибок.
2.  **Завершение работы:** используйте `log.Fatalf` только в функции `main` или `init`. В пакетах возвращайте ошибки через `return err`.
3.  **Контекст:** используйте `SetFlags(log.Lshortfile)`, чтобы видеть, из какого файла и строки была сделана запись.
4.  **Альтернативы:** для сложных проектов (структурированное логирование в формате JSON, уровни логирования) рассмотрите стандартный `log/slog` (начиная с Go 1.21) или сторонние библиотеки (Zap, Logrus).
