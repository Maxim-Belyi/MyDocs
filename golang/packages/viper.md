# Viper

Пакет `Viper` (`github.com/spf13/viper`) — это комплексное решение для конфигурации Go-приложений, созданное для работы в рамках концепции "12-Factor App".

---

## Зачем нужен этот пакет?

Обычно приложения требуют чтения настроек из разных источников:
* Значения по умолчанию
* Файлы конфигурации (JSON, TOML, YAML, HCL, envfile)
* Переменные окружения (Environment Variables)
* Флаги командной строки (Flags)
* Удаленные системы (etcd, Consul)

Viper объединяет все эти источники в единый словарь. Он позволяет разработчику запрашивать параметр (например, `PORT`), а Viper сам решает, откуда его взять, основываясь на приоритетах (флаг > переменная окружения > файл > значение по умолчанию).

---

## Основные методы

### Установка значений и путей

Перед чтением параметров, Viper нужно настроить.

* `viper.SetDefault(key, value)`: устанавливает значение по умолчанию.
* `viper.SetConfigName(name)`: имя файла конфигурации (без расширения).
* `viper.SetConfigType(type)`: расширение файла (например, "yaml").
* `viper.AddConfigPath(path)`: пути, где искать файл конфигурации.

```go
package main

import (
    "fmt"
    "github.com/spf13/viper"
)

func initConfig() {
    // 1. Устанавливаем значения по умолчанию
    viper.SetDefault("port", 8080)
    
    // 2. Настраиваем поиск файла конфигурации
    viper.SetConfigName("config") // имя файла
    viper.SetConfigType("yaml")   // тип файла
    viper.AddConfigPath(".")      // искать в текущей директории
    viper.AddConfigPath("$HOME/.myapp") // и в домашней
}
```

### viper.ReadInConfig()
Выполняет поиск и чтение файла конфигурации на основе ранее заданных настроек.

```go
err := viper.ReadInConfig()
if err != nil {
    panic(fmt.Errorf("Ошибка чтения конфига: %w", err))
}
```

### Интеграция с переменными окружения

Viper может автоматически подхватывать переменные окружения, соответствующие ключам конфигурации.

* `viper.AutomaticEnv()`: включает автоматическое чтение переменных окружения.
* `viper.SetEnvPrefix("APP")`: задает префикс для переменных, чтобы избежать конфликтов в ОС. Теперь Viper будет искать `APP_PORT` вместо просто `PORT`.

```go
viper.SetEnvPrefix("MYAPP")
viper.AutomaticEnv() // Автоматически подхватит MYAPP_PORT
```

### Методы получения значений (Get)

Viper предоставляет методы для получения параметров с приведением к нужному типу: `GetString`, `GetInt`, `GetBool`, `GetDuration`, `GetStringSlice` и т.д.

```go
port := viper.GetInt("port")
dbHost := viper.GetString("database.host") // доступ ко вложенным полям через точку
```

### viper.Unmarshal
Самый популярный метод для работы с конфигурацией в крупных проектах. Вместо того, чтобы получать каждый ключ отдельно через `Get...`, вы можете распарсить всю конфигурацию в структуру Go.

```go
type Config struct {
    Port int
    Database struct {
        Host string
        User string
    }
}

var cfg Config
err := viper.Unmarshal(&cfg)
if err != nil {
    log.Fatalf("unable to decode into struct, %v", err)
}
```

### Привязка конкретных переменных: BindEnv
Если вам не нужно загружать все переменные окружения через `AutomaticEnv`, вы можете точечно привязать ключ конфигурации к конкретной переменной.

```go
// Значение viper.Get("port") будет браться из переменной окружения PORT
viper.BindEnv("port", "PORT")
```

### Сохранение конфигурации: WriteConfig
Viper умеет не только читать, но и сохранять текущие настройки обратно в файл. Это полезно для утилит командной строки (CLI), которые сохраняют настройки пользователя.

```go
// Сохранить в файл, который был загружен
err := viper.WriteConfig()

// Сохранить в конкретный новый файл
err := viper.WriteConfigAs("/path/to/new_config.yaml")
```

### Горячая перезагрузка: WatchConfig
Viper может следить за изменениями файла конфигурации в реальном времени и применять их без перезапуска приложения (hot reload).

```go
import "github.com/fsnotify/fsnotify"

viper.WatchConfig()
viper.OnConfigChange(func(e fsnotify.Event) {
    fmt.Println("Файл конфигурации изменился:", e.Name)
    // Здесь можно обновить внутренние переменные приложения
})
```

## Итог: Чек-лист по пакету Viper

1. **Многослойность:** используйте Viper, если ваше приложение сложнее простого скрипта и требует загрузки настроек как из файлов (YAML/JSON), так и из переменных окружения.
2. **Типизация:** вместо разрозненных вызовов `viper.GetString("key")` по всему коду, создайте единую структуру и используйте `viper.Unmarshal(&cfg)`, передавая эту структуру в качестве зависимости в другие части программы.
3. **Вложенность:** используйте точечную нотацию (`"db.port"`) для доступа к вложенным параметрам конфигурации.
