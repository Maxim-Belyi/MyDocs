IO

Пакет io предоставляет базовые интерфейсы для потокового ввода-вывода данных. Это фундамент, на котором построена работа с файлами, сетевыми соединениями и буферами в Go.

1. Ключевые интерфейсы: Reader и Writer​

Вся работа с данными в пакете io строится вокруг двух небольших интерфейсов.

io.Reader​

Описывает возможность чтения потока байтов. Метод Read читает данные в переданный срез (slice) p и возвращает количество прочитанных байтов и возможную ошибку.

type Reader interface {
    Read(p []byte) (n int, err error)
}

io.EOF (End of File) — это стандартная ошибка, которая возвращается, когда поток данных закончился.

io.Writer​

Описывает возможность записи потока байтов. Метод Write записывает данные из среза p в базовый поток данных.

type Writer interface {
    Write(p []byte) (n int, err error)
}

Поскольку файлы (os.File), сетевые соединения (net.Conn) и буферы (bytes.Buffer) реализуют эти интерфейсы, вы можете писать функции, которые работают с любым источником или приемником данных.

2. Копирование данных: io.Copy​

Функция io.Copy копирует данные из Reader в Writer, пока не будет достигнут конец файла (EOF) или не произойдет ошибка. Она работает с буфером, поэтому не загружает весь файл в оперативную память.

package main

import (
    "io"
    "os"
)

func main() {
    file, _ := os.Open("source.txt")
    defer file.Close()

    // Копируем содержимое файла в стандартный вывод (консоль)
    io.Copy(os.Stdout, file)
}

3. Чтение всего содержимого: io.ReadAll​

Функция io.ReadAll (ранее ioutil.ReadAll) читает данные из Reader до тех пор, пока не встретит EOF или ошибку, и возвращает их в виде массива байтов.

package main

import (
    "fmt"
    "io"
    "strings"
)

func main() {
    reader := strings.NewReader("Текст для чтения")
    
    data, err := io.ReadAll(reader)
    if err != nil {
        panic(err)
    }
    
    fmt.Println(string(data))
}

Внимание: Используйте ReadAll только для небольших объемов данных. Для больших файлов это приведет к переполнению памяти.

4. Продвинутые функции потоков​

io.LimitReader​

Оборачивает существующий Reader и позволяет прочитать из него не более N байт. Это критически важно при чтении данных из ненадежных источников (например, HTTP-запросов от пользователей), чтобы предотвратить атаки на исчерпание памяти (OOM).

// Читаем из тела запроса максимум 1 Мегабайт
r := io.LimitReader(req.Body, 1024*1024)
data, err := io.ReadAll(r)

io.MultiReader​

Позволяет логически объединить несколько интерфейсов Reader в один последовательный. Когда заканчиваются данные в первом, чтение автоматически переходит ко второму, и так далее. Полезно для склеивания файлов "на лету" без их загрузки в память.

r1 := strings.NewReader("Привет, ")
r2 := strings.NewReader("мир!")
r := io.MultiReader(r1, r2) // r при чтении выдаст "Привет, мир!"

io.TeeReader​

"Перехватывает" всё, что читается из Reader, и параллельно записывает это в Writer. Полезно, например, для вычисления хеша (MD5/SHA) скачиваемого файла в процессе его сохранения на диск.

5. Продвинутые интерфейсы: ReadCloser, ReadWriter​

Пакет io комбинирует базовые интерфейсы для описания более сложного поведения.

  • io.ReadCloser: объединяет Reader и метод Close(). Широко используется при работе с HTTP-ответами (resp.Body).
  • io.ReadWriter: объединяет Reader и Writer.
  • io.Seeker: позволяет перемещать указатель позиции чтения/записи.

Итог: Чек-лист по пакету io​

  1. Абстракция: пишите функции, принимающие io.Reader и io.Writer, а не конкретные типы вроде os.File.
  2. Потоковая передача: используйте io.Copy для передачи больших данных без расхода памяти.
  3. Ограничение памяти: избегайте io.ReadAll, если размер входящих данных непредсказуем.
  4. Управление ресурсами: всегда вызывайте метод Close() (обычно через defer) для объектов, реализующих io.Closer.
