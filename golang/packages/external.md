# Внешние пакеты

Хотя стандартная библиотека Go покрывает большинство базовых потребностей, экосистема предоставляет множество проверенных сторонних пакетов для решения специализированных или сложных задач разработки.

---

## 1. Роутеры и веб-фреймворки

Стандартный `net/http` надежен, но требует написания дополнительного кода для работы с переменными пути (path variables) и сложными маршрутами.

### Chi (`github.com/go-chi/chi`)
Легковесный и идиоматичный маршрутизатор. Работает напрямую со стандартным `http.Handler`, что делает его полностью совместимым с любым middleware из экосистемы Go.

```go
r := chi.NewRouter()
r.Get("/users/{id}", getUserHandler)
```

### Gorilla Mux (`github.com/gorilla/mux`)
Исторически один из самых популярных роутеров. Имеет богатый функционал, включая проверку регулярных выражений в URL. Сейчас проект переведен в режим архива (read-only), поэтому новые проекты чаще выбирают Chi или стандартный роутер (который получил поддержку переменных пути в Go 1.22).

---

## 2. Работа с базами данных

Пакет `database/sql` требует ручного маппинга строк из базы данных в структуры Go, что приводит к написанию шаблонного кода.

### sqlx (`github.com/jmoiron/sqlx`)
Расширение для `database/sql`. Позволяет автоматически заполнять поля структур данными из SQL-запроса на основе тегов (например, `db:"user_id"`). Это ускоряет разработку без использования тяжеловесных ORM.

```go
var users []User
err := db.Select(&users, "SELECT * FROM users ORDER BY name ASC")
```

### GORM (`gorm.io/gorm`)
Полноценная ORM для Go. Поддерживает автоматические миграции, связи между таблицами (has one, has many) и сложные построители запросов. Подходит для проектов с сложной бизнес-логикой и большим количеством таблиц.

## 3. Логирование

Стандартный пакет `log` выводит простой текст, что неудобно для автоматизированного сбора логов (Elasticsearch, Kibana).

### Zap (`go.uber.org/zap`)
Разработан в Uber. Отличается высокой производительностью и низким потреблением памяти. Выводит логи в структурированном JSON формате.

```go
logger, _ := zap.NewProduction()
defer logger.Sync()

logger.Info("failed to fetch URL",
    zap.String("url", url),
    zap.Int("attempt", 3),
    zap.Duration("backoff", time.Second),
)
```

### Logrus (`github.com/sirupsen/logrus`)
Популярный логгер с удобным API. Позволяет добавлять поля к сообщениям и поддерживает хуки для отправки логов во внешние системы. На данный момент находится в режиме поддержки (maintenance mode), новые проекты часто используют Zap или стандартный `log/slog`.

## 4. Конфигурация и переменные окружения

### godotenv (`github.com/joho/godotenv`)
Позволяет загружать переменные окружения из файла `.env` в локальной среде разработки. Это стандартный способ разделения конфигурации и кода (по методологии 12-Factor App).

```go
err := godotenv.Load()
if err != nil {
    log.Fatal("Error loading .env file")
}
dbUrl := os.Getenv("DATABASE_URL")
```

### caarlos0/env (`github.com/caarlos0/env`)
Удобный пакет для парсинга переменных окружения в структуры Go. Позволяет задавать значения по умолчанию и обязательные поля прямо через теги структур, избавляя от необходимости писать множество `os.Getenv` и вручную конвертировать строки в нужные типы.

```go
type Config struct {
    Port     int    `env:"PORT" envDefault:"8080"`
    Password string `env:"DB_PASSWORD,unset"`
    IsDebug  bool   `env:"DEBUG"`
}

func main() {
    cfg := Config{}
    if err := env.Parse(&cfg); err != nil {
        log.Fatalf("Ошибка парсинга конфигурации: %v", err)
    }
}
```

### Viper (`github.com/spf13/viper`)
Мощное решение для конфигурации. Умеет читать настройки из файлов (JSON, YAML, TOML), переменных окружения, флагов командной строки и удаленных систем (etcd, Consul), объединяя их в единый объект.

## Итог: Чек-лист по выбору пакетов

1.  **Стандартная библиотека:** всегда проверяйте, можно ли решить задачу силами стандартной библиотеки, прежде чем добавлять зависимость.
2.  **Активность:** выбирайте пакеты, которые активно поддерживаются сообществом (проверяйте дату последних коммитов).
3.  **Стабильность:** избегайте пакетов в статусе "archived" для новых проектов.
4.  **Размер абстракции:** предпочитайте легковесные обертки (например, `sqlx`, `chi`) тяжеловесным фреймворкам, если не уверены, что вам нужен весь функционал последних.
