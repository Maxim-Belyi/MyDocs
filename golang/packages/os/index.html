os

Пакет os предоставляет платформонезависимый интерфейс для работы с функционалом операционной системы. Он используется для работы с файлами, директориями, переменными окружения и процессами.

1. Работа с файлами и директориями​

Пакет позволяет выполнять базовые операции с файловой системой.

Чтение и запись файлов​

Функции os.ReadFile и os.WriteFile — это самые простые способы прочитать файл целиком или записать данные в файл одной строкой кода.

package main

import "os"

func main() {
    // Запись строки в файл
    data := []byte("Пример текста")
    _ = os.WriteFile("test.txt", data, 0644)

    // Чтение всего файла
    content, _ := os.ReadFile("test.txt")
    println(string(content))
}

Управление директориями​

Для создания папок используются функции Mkdir и MkdirAll (создает все промежуточные папки, аналогично mkdir -p в Linux).

// Создание директории с правами доступа 0755
_ = os.MkdirAll("app/logs/2024", 0755)

Открытие и создание файлов​

Для более сложной работы с файлами (построчное чтение, потоковая запись, дозапись в конец) используются функции, возвращающие объект *os.File, который реализует интерфейсы io.Reader и io.Writer.

  • os.Open(name): открывает файл только для чтения.
  • os.Create(name): создает новый файл (или очищает существующий) для чтения и записи.
  • os.OpenFile(name, flag, perm): универсальная функция. Позволяет указать флаги (например, os.O_APPEND для дозаписи в конец) и права доступа.
// Открытие файла для дозаписи логов
file, err := os.OpenFile("app.log", os.O_APPEND|os.O_CREATE|os.O_WRONLY, 0644)
if err != nil {
    log.Fatal(err)
}
defer file.Close()

Получение информации о файле: Stat​

Функция os.Stat возвращает структуру os.FileInfo с метаданными файла (размер, дата изменения, является ли папкой). Часто используется для проверки существования файла:

if _, err := os.Stat("config.json"); os.IsNotExist(err) {
    fmt.Println("Файл не существует")
}

2. Работа с переменными окружения​

Переменные окружения — стандартный способ передачи конфигурации приложению.

package main

import (
    "fmt"
    "os"
)

func main() {
    // Установка переменной
    os.Setenv("PORT", "8080")

    // Получение переменной
    port := os.Getenv("PORT")
    
    // Получение с проверкой существования переменной
    host, exists := os.LookupEnv("DB_HOST")
    if !exists {
        fmt.Println("DB_HOST не задан, используем значение по умолчанию")
        host = "localhost"
    }
    
    fmt.Printf("Подключение к %s:%s\n", host, port)
}

3. Завершение работы и аргументы командной строки​

os.Exit​

Функция os.Exit немедленно завершает программу с указанным кодом возврата (0 означает успешное завершение). Важно помнить, что при её вызове функции, запланированные через defer, не выполняются.

if err != nil {
    // Программа немедленно завершится, defer не сработают
    os.Exit(1)
}

os.Args​

Срез os.Args содержит аргументы командной строки, переданные при запуске программы. Нулевой элемент (os.Args[0]) — это путь к самому исполняемому файлу.

// Запуск: go run main.go -config=config.yml
func main() {
    if len(os.Args) > 1 {
        fmt.Println("Передан аргумент:", os.Args[1])
    }
}

Для сложного парсинга флагов командной строки рекомендуется использовать стандартный пакет flag или внешние библиотеки, а не обрабатывать os.Args вручную.

4. Сигналы ОС (os/signal)​

Пакет os/signal позволяет перехватывать системные сигналы (например, когда пользователь нажимает Ctrl+C, отправляя сигнал SIGINT, или когда Kubernetes просит контейнер завершиться через SIGTERM).

С версии Go 1.16 для этого рекомендуется использовать signal.NotifyContext — это самый простой и современный способ реагировать на сигналы.

package main

import (
    "context"
    "fmt"
    "os/signal"
    "syscall"
)

func main() {
    // Создаем контекст, который будет отменен при получении SIGINT или SIGTERM
    ctx, stop := signal.NotifyContext(context.Background(), syscall.SIGINT, syscall.SIGTERM)
    defer stop()

    fmt.Println("Сервер работает. Нажмите Ctrl+C для остановки...")
    
    // Блокируем выполнение, пока контекст не будет отменен
    <-ctx.Done()
    
    fmt.Println("\nПолучен сигнал остановки! Завершаем работу.")
}

Сигнал SIGKILL (kill -9) перехватить невозможно — операционная система убивает процесс мгновенно.

Итог: Чек-лист по пакету os​

  1. Файлы: используйте os.ReadFile и os.WriteFile для простых операций. Для потоковой обработки больших файлов открывайте их через os.Open и используйте пакет io.
  2. Переменные окружения: используйте os.LookupEnv, если вам нужно различать пустую строку от отсутствующей переменной.
  3. Остановка: избегайте os.Exit глубоко в коде. Возвращайте ошибки наверх и завершайте программу только в функции main.
  4. Права доступа: при создании файлов и папок указывайте стандартные Unix-права (например, 0644 для файлов, 0755 для директорий).
  5. Сигналы: всегда используйте signal.NotifyContext для перехвата сигналов ОС и реализации Graceful Shutdown, не полагаясь на старые методы с каналами.
