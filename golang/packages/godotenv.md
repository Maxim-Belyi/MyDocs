# godotenv

Пакет `godotenv` (от `github.com/joho/godotenv`) — это порт популярной Ruby-библиотеки dotenv для Go. Он предназначен для загрузки переменных окружения из файла `.env` в среду выполнения процесса (environment). 

---

## Зачем нужен этот пакет?

Согласно методологии "Приложение двенадцати факторов" (12-Factor App), конфигурация должна храниться в переменных окружения. При локальной разработке неудобно каждый раз задавать десятки переменных в консоли. Файл `.env` позволяет хранить эти переменные локально (и не коммитить его в Git). 

`godotenv` читает этот файл и применяет переменные в систему так, будто они были экспортированы в консоли.

---

## Основные методы

### godotenv.Load
Загружает переменные из файла `.env` (в текущей директории). Если файл не найден, возвращает ошибку.

```go
func Load(filenames ...string) error
```

**Пример использования:**
```go
package main

import (
    "log"
    "os"
    "github.com/joho/godotenv"
)

func main() {
    // Загрузка переменных из .env
    err := godotenv.Load()
    if err != nil {
        log.Fatal("Ошибка загрузки файла .env")
    }

    // Теперь можно использовать стандартный пакет os
    dbHost := os.Getenv("DB_HOST")
    log.Println("Подключение к:", dbHost)
}
```

Можно передать несколько файлов. Они будут загружаться по порядку. Переменные, загруженные раньше, **не перезаписываются** переменными из последующих файлов.
```go
godotenv.Load(".env.local", ".env")
```

### godotenv.Overload
Работает точно так же, как `Load`, но **перезаписывает** уже существующие системные переменные окружения, если они совпадают с ключами из файла `.env`. Полезно, когда значения из `.env` должны иметь высший приоритет.

```go
func Overload(filenames ...string) error
```

### godotenv.Read
Возвращает переменные в виде словаря `map[string]string`, но **не применяет** их в системные переменные окружения.

```go
func Read(filenames ...string) (envMap map[string]string, err error)
```

**Пример использования:**
```go
envMap, err := godotenv.Read(".env")
if err != nil {
    log.Fatal(err)
}

fmt.Println("Значение DB_PORT:", envMap["DB_PORT"])
```

### godotenv.Parse
Парсит строку или поток (реализующий `io.Reader`) с форматом dotenv и возвращает словарь `map[string]string`. Полезно, если данные приходят не из файла (например, из сети).

```go
func Parse(r io.Reader) (envMap map[string]string, err error)
```

### godotenv.Write
Записывает словарь `map[string]string` обратно в файл в формате dotenv.

```go
func Write(envMap map[string]string, filename string) error
```

### godotenv.Unmarshal / godotenv.Marshal
Используются для конвертации между форматом `.env` (текст) и `map[string]string` в памяти. Работают аналогично функциям пакета `json`.

## Итог: Чек-лист по пакету godotenv

1. **Только для разработки:** в production-окружении (например, в Kubernetes или Docker) переменные окружения обычно задаются самой платформой. В таких случаях вызов `Load` может возвращать ошибку отсутствия файла, которую нужно игнорировать, либо использовать `godotenv` только при локальном запуске.
2. **Связка с другими пакетами:** `godotenv` отлично работает в связке с `caarlos0/env`. Сначала вы вызываете `godotenv.Load()`, а затем `env.Parse(&cfg)`.
3. **Безопасность:** никогда не добавляйте файлы с секретами (`.env`) в систему контроля версий (Git). Добавляйте их в `.gitignore`. Вы можете создать файл `.env.example` с пустыми значениями для демонстрации структуры конфигурации.
