# context.Context

Пакет `context` — это стандартный инструмент в Go для передачи сигналов отмены, дедлайнов (таймаутов) и специфичных для запроса данных (request-scoped data) сквозь границы API и между горутинами.

---

## 1. Базовые принципы контекста

Контекст должен передаваться первым параметром в любую функцию, которая может выполняться долго (сетевые запросы, обращения к БД) или запускает новые горутины. По конвенции этот параметр всегда называется `ctx`.

```go
// Правильная сигнатура функции
func DoSomething(ctx context.Context, arg1 string) error {
    // ...
}
```

Для создания пустых контекстов используются:
*   `context.Background()`: корневой контекст. Используется в функции `main`, инициализации или тестах.
*   `context.TODO()`: используется, когда неясно, какой контекст нужен, или если функция еще не принимает контекст, но будет принимать в будущем.

---

## 2. Отмена операций: WithCancel

Контекст с отменой используется для того, чтобы остановить горутины, когда их работа больше не нужна.

```go
package main

import (
    "context"
    "fmt"
    "time"
)

func main() {
    // Создаем контекст и функцию отмены
    ctx, cancel := context.WithCancel(context.Background())
    
    // Запускаем горутину
    go worker(ctx)

    // Даем ей поработать 2 секунды
    time.Sleep(2 * time.Second)
    
    // Отменяем контекст
    cancel()
    
    // Ждем немного перед выходом
    time.Sleep(1 * time.Second)
}

func worker(ctx context.Context) {
    for {
        select {
        case <-ctx.Done(): // Срабатывает при вызове cancel()
            fmt.Println("Работа остановлена")
            return
        default:
            fmt.Println("Выполнение работы...")
            time.Sleep(500 * time.Millisecond)
        }
    }
}
```

---

## 3. Таймауты и дедлайны: WithTimeout

Используется для ограничения времени выполнения операции (например, запроса к базе данных).

```go
// Выделяем 3 секунды на выполнение задачи
ctx, cancel := context.WithTimeout(context.Background(), 3*time.Second)
defer cancel() // Обязательно вызываем cancel, даже если задача завершилась досрочно

// Передаем контекст в функцию (например, HTTP-запрос или SQL-запрос)
err := db.QueryRowContext(ctx, "SELECT * FROM users").Scan(&user)
if err != nil {
    // Если пройдет 3 секунды, база данных прервет запрос и вернет ошибку context.DeadlineExceeded
}
```

> **Важно:** Вызов `cancel()` в связке с `defer` необходим для немедленного освобождения ресурсов, связанных с контекстом, не дожидаясь истечения таймаута.

---

## 4. Передача данных: WithValue

Контекст можно использовать для передачи данных, специфичных для запроса (например, ID пользователя из токена авторизации).

```go
// Создаем тип-ключ, чтобы избежать коллизий со встроенными типами
type contextKey string
const userIDKey contextKey = "userID"

// Запись в контекст
ctx := context.WithValue(context.Background(), userIDKey, 42)

// Чтение из контекста
if val := ctx.Value(userIDKey); val != nil {
    fmt.Printf("User ID: %v\n", val)
}
```

> Использовать `WithValue` следует только для опциональных транзитных данных запроса (request-scoped data). Никогда не передавайте через контекст обязательные параметры бизнес-логики или зависимости (например, логгеры или подключения к БД).

---

## Итог: Чек-лист по пакету context

1.  **Позиция:** `ctx context.Context` всегда должен быть первым аргументом функции.
2.  **Структуры:** не сохраняйте `context` в полях структур. Передавайте его явно в вызовы методов.
3.  **Корневой контекст:** используйте `context.Background()` в качестве отправной точки на самом верхнем уровне.
4.  **Утечки памяти:** всегда вызывайте функцию `cancel()` после `WithCancel` или `WithTimeout`, даже если операция завершилась успешно. Обычно это делается через `defer cancel()`.
5.  **Данные:** используйте `WithValue` экономно и только для метаданных запроса (ID трейсинга, авторизация).
