# Or-Done (Канал завершения)

При работе с каналами в Go часто возникает ситуация, когда вам нужно читать из канала в цикле `for range`, но вы **не можете гарантировать**, что этот канал когда-нибудь будет закрыт его создателем.

Если канал никогда не закроют, цикл `for range` заблокируется навечно (Goroutine Leak). 

Обычно эту проблему решают с помощью блока `select` и дополнительного канала отмены `done` (или контекста). Но если вам нужно читать данные из канала глубоко внутри вашей бизнес-логики, постоянное написание громоздких конструкций с `select` делает код нечитаемым.

Паттерн **Or-Done** оборачивает эту логику в удобную функцию.

---

## 1. Проблема

Мы хотим читать из канала просто:
```go
for val := range myStream {
    fmt.Println(val)
}
```

Но безопасный (отменяемый) вариант требует писать так:
```go
loop:
for {
    select {
    case <-done:
        break loop
    case val, ok := <-myStream:
        if !ok {
            break loop
        }
        fmt.Println(val)
    }
}
```
Это слишком многословно.

---

## 2. Реализация Or-Done

Мы создаем функцию-хелпер `orDone`, которая принимает канал отмены `done` и целевой канал `c`. Она возвращает **новый канал**, из которого можно безопасно читать через простой `for range`. Новый канал закроется либо когда закроется `c`, либо когда закроется `done`.

```go
package main

import "fmt"

// orDone возвращает прокси-канал для безопасного чтения
func orDone(done <-chan struct{}, c <-chan interface{}) <-chan interface{} {
	valStream := make(chan interface{})
	
	go func() {
		defer close(valStream) // Гарантируем закрытие прокси-канала
		
		for {
			select {
			case <-done:
				// Сигнал отмены (прекращаем работу)
				return
			case v, ok := <-c:
				if !ok {
					// Исходный канал закрыт (прекращаем работу)
					return
				}
				
				// Пытаемся отправить данные в прокси-канал.
				// Важно: здесь тоже нужен select с done, так как 
				// потребитель (кто читает из valStream) мог уже отключиться!
				select {
				case valStream <- v:
				case <-done:
				}
			}
		}
	}()
	
	return valStream
}
```

## 3. Использование паттерна

Теперь вы можете использовать обычный, красивый `for range` цикл, не беспокоясь о зависании, даже если вы работаете с неизвестным каналом из сторонней библиотеки:

```go
func processData(done <-chan struct{}, unknownStream <-chan interface{}) {
	
	// Оборачиваем неизвестный канал в orDone
	for val := range orDone(done, unknownStream) {
		
		// Если кто-то закроет done-канал, этот цикл 
		// гарантированно завершится, и утечки горутины не произойдет.
		fmt.Printf("Обработка: %v\n", val)
		
	}
	fmt.Println("Обработка завершена безопасно.")
}
```

**Резюме:** Паттерн Or-Done (иногда его называют *Done Channel Wrapper*) — это синтаксический сахар для безопасного, прерываемого чтения из каналов.
