# Bridge-Channel (Мост)

 **Bridge (Мост)** используется для решения специфической проблемы в сложных конвейерах: **превращения канала каналов в плоский канал данных**.

Иногда стадия конвейера возвращает не поток элементов, а поток *каналов*, из которых уже можно читать элементы. В Go это выглядит как переменная с типом `<-chan <-chan interface{}`. Читать данные из такой конструкции напрямую очень больно, так как нужно управлять жизненным циклом и вложенными циклами.

Паттерн Bridge берет на себя задачу "сплющивания" (flattening) этой структуры. На выходе он отдает удобный плоский канал `<-chan interface{}`.

---

## Зачем возвращать канал каналов?
Обычно это происходит, когда конвейер имеет динамическое количество источников. Например, вы обходите файловую систему, находите 100 лог-файлов, и для каждого файла запускаете горутину, которая возвращает канал `<-chan string` (строки лога). Чтобы не ждать первый файл целиком, мы кладем сами эти 100 каналов в общий канал-оркестратор.

## Реализация

Суть паттерна — в двух вложенных циклах (часто через `for range`). Внешний цикл читает входящие каналы, а внутренний цикл — читает значения из этих каналов, пока текущий канал не закроется, затем берется следующий канал.

```go
package main

import "fmt"

// Bridge сплющивает канал каналов в один плоский канал.
func Bridge(done <-chan struct{}, chanStream <-chan <-chan interface{}) <-chan interface{} {
	valStream := make(chan interface{})
	
	go func() {
		defer close(valStream)
		
		// 1. Читаем очередной канал (поток) из канала каналов
		for stream := range chanStream {
			
			// 2. Читаем все элементы из текущего извлеченного канала
			for val := range stream {
				select {
				case <-done:
					// Быстрая отмена
					return
				case valStream <- val:
					// Пересылаем в плоский результирующий канал
				}
			}
			// Когда stream закрывается, мы выходим из внутреннего цикла
			// и берем следующий stream из chanStream.
		}
	}()
	
	return valStream
}
```

## Использование

Давайте создадим генератор, который выдает 3 канала, в каждом из которых по 2 элемента.

```go
// genChannels создает канал, в который складывает другие каналы.
func genChannels(done <-chan struct{}) <-chan <-chan interface{} {
	out := make(chan (<-chan interface{}))
	
	go func() {
		defer close(out)
		
		for i := 0; i < 3; i++ {
			// Создаем мелкий канал на 2 элемента
			stream := make(chan interface{}, 2)
			stream <- i * 10
			stream <- (i * 10) + 1
			close(stream) // Закрываем мелкий канал
			
			select {
			case <-done:
				return
			case out <- stream: // Отправляем мелкий канал в общий канал каналов
			}
		}
	}()
	
	return out
}

func main() {
	done := make(chan struct{})
	defer close(done)

	// Получаем канал каналов (chanStream типа <-chan <-chan interface{})
	chanStream := genChannels(done)

	// Прогоняем его через мост, получая плоский поток
	flatStream := Bridge(done, chanStream)

	// Теперь мы можем просто и линейно читать все данные
	for v := range flatStream {
		fmt.Println("Значение:", v)
	}
	
	// Вывод:
	// Значение: 0
	// Значение: 1
	// Значение: 10
	// Значение: 11
	// Значение: 20
	// Значение: 21
}
```

Паттерн Bridge делает потребляющий код (вызывающую сторону) чистым и независимым от того, как внутри конвейера данные были распределены (batching) по под-каналам.
