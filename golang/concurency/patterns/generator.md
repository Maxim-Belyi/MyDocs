# Generator (Генератор)

**Паттерн Generator (Генератор)** — паттерн в Go, который используется для создания потока данных (stream of data).

Суть паттерна: функция инкапсулирует в себе создание канала, запускает горутину, которая пишет в этот канал, и возвращает этот канал наружу (обычно как read-only `<-chan`).

## Зачем это нужно?
Вместо того чтобы заставлять вызывающий код (клиента) создавать каналы, передавать их в функции и заботиться об их закрытии, функция-генератор берет всю ответственность за управление жизненным циклом горутины и канала на себя. Вызывающий код получает готовый поток данных.

## Пример реализации

```go
package main

import (
	"fmt"
	"time"
)

// Generator возвращает канал, из которого можно читать целые числа.
// Мы передаем канал `done` для безопасной отмены (чтобы не было утечки горутин).
func Generator(done <-chan struct{}, integers ...int) <-chan int {
	// 1. Создаем канал
	intStream := make(chan int)

	// 2. Запускаем горутину
	go func() {
		// 3. Гарантируем закрытие канала при завершении горутины
		defer close(intStream)

		for _, i := range integers {
			select {
			case <-done:
				// Если пришел сигнал отмены, прекращаем работу
				return
			case intStream <- i:
				// Иначе отправляем данные в канал
			}
		}
	}()

	// 4. Возвращаем канал (приведенный к read-only)
	return intStream
}

func main() {
	done := make(chan struct{})
	defer close(done) // Освобождаем ресурсы при выходе из main

	// Вызываем генератор
	intStream := Generator(done, 1, 2, 3, 4, 5)

	// Читаем из сгенерированного канала
	for v := range intStream {
		fmt.Printf("Получено: %d\n", v)
		time.Sleep(100 * time.Millisecond)
	}
}
```

## Важные моменты

1. **Защита от утечек (Goroutine Leaks):** Всегда передавайте канал `done` (или `context.Context`) в генератор. Если потребитель (consumer) перестанет читать из канала `intStream` до того, как генератор отправит все данные, горутина генератора заблокируется на записи (`intStream <- i`) навсегда. Канал `done` позволяет подать сигнал горутине завершиться.
2. **Инкапсуляция состояния:** Переменные, с которыми работает горутина, безопасно скрыты внутри функции. Вызывающий код не имеет к ним прямого доступа, что исключает состояние гонки (Race Condition).
3. **Основа для пайплайнов:** Генераторы обычно служат первой (инициирующей) стадией в паттерне Pipeline.
