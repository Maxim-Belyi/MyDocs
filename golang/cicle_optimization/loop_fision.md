# Loop Fission (Расщепление) 


Расщепление применяется для борьбы с **Cache Thrashing (Переполнением кэша)** и попыткой подсказать компилятору использовать SIMD-инструкции.

Давайте разберем эти подходы на основе реальных бенчмарков (CPU: AMD Ryzen 5 7500F).

---

## Сценарий 1: Победа над Cache Thrashing (Давление на кэш)

Представим тяжелую математическую задачу, где внутри одного цикла мы одновременно читаем и пишем в **6 разных огромных массивов**. 

Каждый массив в нашем тесте содержит 1 миллион элементов типа `int` (8 байт), то есть весит **~8 Мегабайт**. Суммарно в одном цикле мы пытаемся жонглировать **48 Мегабайтами** данных.

**Единый цикл:**
```go
// BenchmarkSingleBigLoop (6 массивов в одном цикле = 48 МБ)
for j := 0; j < size; j++ {
    A[j] = B[j] + C[j]
    D[j] = E[j] * F[j]
}
```

**Расщепленный цикл:**
```go
// BenchmarkFissionLoops (По 3 массива на цикл = 24 МБ)
for j := 0; j < size; j++ {
    A[j] = B[j] + C[j]
}
for j := 0; j < size; j++ {
    D[j] = E[j] * F[j]
}
```

### Результаты бенчмарка:
*   Единый цикл: `1,460,997 ns/op`
*   Расщепленные циклы: `1,025,024 ns/op`
*   **Ускорение: ~30% (почти на полмиллисекунды быстрее!)**

**Разбор полетов: **
Почему расщепление ускорило код, если мы добавили "лишний" проход цикла? Секрет кроется в характеристиках процессора Ryzen 5 7500F. 
Объем его сверхбыстрого **L3-кэша составляет 32 Мегабайта**. 
*   В первом тесте мы работаем с 48 МБ данных одновременно. Они физически **не помещаются** в кэш. Процессор начинает судорожно вытеснять данные из кэша в оперативную память и обратно (это и называется Cache Thrashing). 
*   Разбив цикл на два, мы ограничили "рабочий набор" данных до **24 МБ**. Теперь все три массива для первой половины вычислений **идеально поместились в L3-кэш процессора**! Отработав с ними на максимальной скорости, процессор берется за вторые 24 МБ.

## Сценарий 2: Ловушка SIMD в компиляторе Go (Unexpected Failure)

В языках вроде C++ и Rust (которые используют компиляторы GCC или LLVM) есть мощнейшая фича — **Авто-векторизация**. Если компилятор видит простой математический цикл без ветвлений (`if/else`), он использует SIMD-инструкции (например, AVX2), чтобы умножать не по одному числу за раз, а сразу по 4 или 8 чисел. Но наличие `if` внутри цикла ломает эту оптимизацию.

Поэтому в теории (в мире C/C++) разбиение цикла на "чистую математику" и "отдельный if" должно дать гигантский прирост скорости.

**Смешанный цикл (Ветвление + Математика):**
```go
// BenchmarkMixedLoop
for j := 0; j < size; j++ {
    data1[j] = data2[j] * 3  // Математика
    if cond[j] == 0 {        // Ветвление
        count++
    }
}
```

**Расщепленный цикл для SIMD:**
```go
// BenchmarkSplitForSIMD
for j := 0; j < size; j++ {
    data1[j] = data2[j] * 3  // Ожидаем, что тут включится SIMD
}
for j := 0; j < size; j++ {
    if cond[j] == 0 {        // Обрабатываем условия отдельно
        count++
    }
}
```

### Результаты бенчмарка:
*   Смешанный цикл (Один проход): `438,946 ns/op`
*   Расщепленный цикл (Два прохода): `654,559 ns/op`
*   **Провал: Расщепление сделало код МЕДЛЕННЕЕ на 49%!**
