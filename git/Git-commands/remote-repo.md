#### ** Работа с удалёнными репозиториями**

*   `git pull`
    **Что делает:** Загружает изменения из удалённого репозитория и автоматически пытается слить их с вашей текущей веткой. По сути, это комбинация двух команд: `git fetch` (загрузить) + `git merge` (слить).

*   `git push`
    **Что делает:** Выгружает ваши локальные коммиты на удалённый репозиторий. Этой командой вы делитесь своими изменениями с другими.
    **Пример:** `git push origin main` — отправить коммиты из локальной ветки `main` в удалённый репозиторий с именем `origin`.

*   `git fetch`
    **Что делает:** Загружает всю историю коммитов и веток из удалённого репозитория, но **не** применяет их к вашим локальным веткам. Это полезно, чтобы посмотреть, какие изменения произошли на сервере, прежде чем сливать их со своей работой.
